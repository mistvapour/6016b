# 🔧 修复 IDE 类型检查器错误

## ❌ 问题描述

IDE（Cursor/VSCode）显示以下类型检查错误：

```
无法解析导入 "fastapi"
无法解析导入 "pydantic"
无法解析导入 "uvicorn"
```

## 🔍 原因

类型检查器（basedpyright）无法找到虚拟环境（`.venv`）中已安装的包。

## ✅ 解决方案

### 方案 1: 配置 IDE 使用正确的 Python 解释器

1. **在 Cursor/VSCode 中**：
   - 按 `Ctrl+Shift+P` 打开命令面板
   - 输入 `Python: Select Interpreter`
   - 选择：`.\backend\.venv\Scripts\python.exe`

### 方案 2: 使用已创建的配置文件

我已经创建了以下配置文件：

1. **`.vscode/settings.json`** - VSCode/Cursor 工作区设置
2. **`backend/pyrightconfig.json`** - Pyright 类型检查器配置
3. **`pyrightconfig.json`** - 项目根目录的全局配置

### 方案 3: 重启 IDE

配置完成后，需要：
1. 重新加载窗口：`Ctrl+Shift+P` → `Developer: Reload Window`
2. 或者完全关闭并重新打开 IDE

### 方案 4: 检查虚拟环境

确保虚拟环境中已安装所需包：

```bash
cd backend
source .venv/Scripts/activate
pip list | grep -E "fastapi|pydantic|uvicorn"
```

如果没有安装，运行：

```bash
pip install -r requirements.txt
```

## 📋 配置文件说明

### `.vscode/settings.json`

```json
{
  "python.defaultInterpreterPath": "${workspaceFolder}/backend/.venv/Scripts/python.exe",
  "basedpyright.analysis.venv": "backend/.venv"
}
```

### `backend/pyrightconfig.json`

```json
{
  "venvPath": ".",
  "venv": ".venv"
}
```

## 🔄 验证修复

配置完成后：

1. **重新加载 IDE 窗口**
2. **检查类型错误是否消失**
3. **验证导入提示是否正常**

## ⚠️ 注意事项

- 确保虚拟环境路径正确：`backend/.venv`
- 确保虚拟环境中已安装所有依赖
- 可能需要重启 IDE 才能生效

## 🎯 快速修复步骤

1. 按 `Ctrl+Shift+P`
2. 选择 `Python: Select Interpreter`
3. 选择 `.\backend\.venv\Scripts\python.exe`
4. 重新加载窗口：`Ctrl+Shift+P` → `Developer: Reload Window`
