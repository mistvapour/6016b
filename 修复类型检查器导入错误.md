# 🔧 修复类型检查器导入错误

## ❌ 错误信息

```
无法解析导入 "fastapi"
无法解析导入 "pydantic"
无法解析导入 "uvicorn"
```

## ✅ 解决方案

### 方法 1: 在 IDE 中选择正确的 Python 解释器（推荐）

1. **打开命令面板**：
   - 按 `Ctrl+Shift+P`（Windows/Linux）
   - 或 `Cmd+Shift+P`（Mac）

2. **选择 Python 解释器**：
   - 输入并选择：`Python: Select Interpreter`
   - 选择：`.\backend\.venv\Scripts\python.exe`

3. **重新加载窗口**：
   - 按 `Ctrl+Shift+P`
   - 输入并选择：`Developer: Reload Window`

### 方法 2: 验证配置文件

我已经创建了以下配置文件：

- **`.vscode/settings.json`** - 已配置指向 `backend/.venv`
- **`backend/pyrightconfig.json`** - 已配置虚拟环境路径

如果错误仍然存在，请：

1. **重启 IDE**（完全关闭并重新打开）
2. **清除缓存**：
   - 删除 `.vscode` 文件夹中的缓存文件（如果有）
   - 或者重新加载窗口

### 方法 3: 手动配置 basedpyright

如果上述方法无效，可以在工作区设置中直接指定：

在 `.vscode/settings.json` 中添加：

```json
{
  "basedpyright.analysis.extraPaths": [
    "${workspaceFolder}/backend/.venv/Lib/site-packages"
  ]
}
```

## 📋 验证步骤

1. ✅ 确认虚拟环境存在：`backend/.venv`
2. ✅ 确认包已安装（已验证）
3. ✅ 选择正确的 Python 解释器
4. ✅ 重新加载 IDE 窗口

## 🎯 快速修复

**一键修复步骤：**

1. `Ctrl+Shift+P` → `Python: Select Interpreter`
2. 选择 `.\backend\.venv\Scripts\python.exe`
3. `Ctrl+Shift+P` → `Developer: Reload Window`

完成！类型检查错误应该消失了。

## ⚠️ 注意事项

- 这些是**警告**，不是错误，不会影响程序运行
- 如果不需要类型检查，可以忽略这些警告
- 或者禁用 basedpyright：在设置中搜索 `basedpyright` 并禁用
