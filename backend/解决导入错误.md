# 🔧 解决 "Could not import module 'main'" 错误

## ❌ 错误信息

```
ERROR: Error loading ASGI app. Could not import module "main".
```

## 🔍 可能的原因

1. **工作目录不对** - uvicorn 不在 backend 目录下运行
2. **模块导入失败** - main.py 在导入时出错
3. **语法错误** - main.py 或依赖模块有语法错误

## ✅ 解决方案

### 方法 1: 确保在正确的目录运行

**确保在 backend 目录下运行 uvicorn：**

```bash
cd backend
source .venv/Scripts/activate
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

**或者使用完整路径：**

```bash
python -m uvicorn backend.main:app --host 127.0.0.1 --port 8000 --reload
```

### 方法 2: 检查模块导入

运行以下命令检查 main.py 是否能正常导入：

```bash
cd backend
python -c "from main import app; print('✓ 导入成功')"
```

如果有错误，查看错误信息。

### 方法 3: 检查语法错误

```bash
cd backend
python -m py_compile main.py
```

### 方法 4: 使用绝对路径

修改启动命令，使用绝对路径：

```bash
cd C:\Users\Administrator\Documents\毕业论文\代码\前端\6016-app\backend
.venv\Scripts\activate
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

## 🎯 快速修复

1. **关闭当前终端窗口**
2. **打开新的终端**
3. **切换到 backend 目录**：
   ```bash
   cd C:\Users\Administrator\Documents\毕业论文\代码\前端\6016-app\backend
   ```
4. **激活虚拟环境**：
   ```bash
   source .venv/Scripts/activate
   ```
5. **启动服务**：
   ```bash
   python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
   ```

## 📋 验证步骤

启动后应该看到：
```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Started reloader process [...]
INFO:     Application startup complete.
```

如果没有看到 "Application startup complete"，说明模块导入还有问题。
