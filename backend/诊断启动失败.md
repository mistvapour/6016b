# 🔍 诊断服务启动失败

## 问题现象

看到这些日志：
```
INFO: Uvicorn running on http://127.0.0.1:8000
INFO: Started reloader process [91700] using WatchFiles
```

但服务无法访问，说明：
- ✅ Uvicorn 服务器已启动
- ❌ FastAPI 应用启动失败

## 可能的原因

1. **应用导入时出错** - 某个模块导入失败
2. **路由注册失败** - 某个路由定义有问题
3. **数据库连接失败** - 虽然 db 模块可用，但连接时出错
4. **API 模块初始化失败** - 某个 API 模块初始化出错

## 诊断步骤

### 步骤 1: 查看完整错误信息

在运行服务的窗口中，查看是否有红色错误信息。常见错误：

- `ModuleNotFoundError` - 缺少模块
- `ImportError` - 导入失败
- `ConnectionError` - 连接失败
- `SyntaxError` - 语法错误

### 步骤 2: 尝试不启用 reload 启动

```bash
python -m uvicorn main:app --host 127.0.0.1 --port 8000
```

移除 `--reload` 参数，看看是否能正常启动。

### 步骤 3: 检查是否有语法错误

```bash
python -m py_compile main.py
```

### 步骤 4: 查看应用启动日志

在服务启动的窗口中，应该会看到：
- `Application startup complete.` - 表示启动成功
- 如果有错误，会显示具体的错误信息

## 常见错误和解决方案

### 错误 1: 数据库连接失败

**错误信息：**
```
mysql.connector.errors.InterfaceError: Can't connect to MySQL server
```

**解决方案：**
- 检查数据库服务是否运行
- 检查 `.env` 文件中的数据库配置
- 或者临时禁用数据库相关功能

### 错误 2: 模块导入失败

**错误信息：**
```
ModuleNotFoundError: No module named 'xxx'
```

**解决方案：**
```bash
pip install -r requirements.txt
```

### 错误 3: 语法错误

**错误信息：**
```
SyntaxError: invalid syntax
```

**解决方案：**
- 检查 main.py 或相关文件的语法
- 使用 `python -m py_compile` 检查
