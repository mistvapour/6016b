# ✅ 启动服务的正确方法

## ❌ 错误的方式

**在项目根目录运行 uvicorn：**
```bash
cd C:\Users\Administrator\Documents\毕业论文\代码\前端\6016-app
python -m uvicorn main:app  # ❌ 错误！找不到 main.py
```

**结果：**
```
ERROR: Error loading ASGI app. Could not import module "main".
```

## ✅ 正确的方式

### 方法 1: 在 backend 目录下运行（推荐）

```bash
# 步骤 1: 切换到 backend 目录
cd C:\Users\Administrator\Documents\毕业论文\代码\前端\6016-app\backend

# 步骤 2: 激活虚拟环境
source .venv/Scripts/activate

# 步骤 3: 启动服务
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

### 方法 2: 在项目根目录使用完整路径

```bash
# 在项目根目录
cd C:\Users\Administrator\Documents\毕业论文\代码\前端\6016-app

# 使用完整路径
python -m uvicorn backend.main:app --host 127.0.0.1 --port 8000 --reload
```

## 🔍 如何确认在正确的目录

启动前运行：

```bash
pwd
ls main.py
```

如果：
- ✅ 当前目录是 `backend`
- ✅ 能看到 `main.py` 文件

说明在正确的位置。

## ✅ 成功启动的标志

看到以下输出说明启动成功：

```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Started reloader process [进程ID] using WatchFiles
INFO:     Application startup complete.
```

## 📋 总结

- ❌ **不要在项目根目录运行** `uvicorn main:app`
- ✅ **必须在 backend 目录运行**，或使用 `backend.main:app`
- ✅ **确保虚拟环境已激活**
- ✅ **使用 `--host 127.0.0.1` 和 `--port 8000`**
