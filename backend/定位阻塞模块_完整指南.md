# 🔍 定位阻塞模块 - 完整指南

## 🎯 问题分析

根据经验，**99% 的卡住问题**出在模块导入时：

1. **循环导入** - 模块 A 导入模块 B，模块 B 又导入模块 A
2. **导入时执行阻塞代码** - 模块在导入时就连接数据库/网络
3. **导入超时** - 模块在导入时尝试连接外部服务

## 🚀 诊断步骤

### 步骤 1: 使用最小版本测试（最快）

**运行最小版本**：
```bash
cd backend
source .venv/Scripts/activate
python -m uvicorn main_最小版:app --host 127.0.0.1 --port 8000
```

**如果最小版本能快速启动**：
- ✅ 说明基础配置没问题
- ✅ 问题出在被注释掉的某个模块
- ✅ 继续步骤 2

**如果最小版本也卡住**：
- ❌ 可能是数据库连接问题
- ❌ 检查数据库服务是否运行

### 步骤 2: 使用主版本查看日志

**运行主版本（已添加详细日志）**：
```bash
python -m uvicorn main:app --host 127.0.0.1 --port 8000
```

**观察日志输出**：
- 查看最后一行打印的是哪个模块
- **卡住前最后显示的模块就是问题模块**

例如，如果看到：
```
>>> 正在加载 MQTT API 模块...
```

然后卡住不动，说明 `mqtt_api.py` 有问题。

### 步骤 3: 逐个模块测试

在 `main.py` 中，逐个取消注释模块，每次测试一个：

**方法 1: 修改 main.py**

注释掉所有模块，然后逐个取消注释：

```python
# 先注释掉所有
PDF_API_AVAILABLE = False
MQTT_API_AVAILABLE = False
# ...

# 然后测试第一个
print(">>> 正在加载 PDF API 模块...", file=sys.stderr)
try:
    from pdf_api import include_pdf_routes
    PDF_API_AVAILABLE = True
    print("✅ PDF API模块加载成功", file=sys.stderr)
except ImportError as e:
    print(f"❌ PDF API模块导入失败: {e}", file=sys.stderr)
    PDF_API_AVAILABLE = False
```

**方法 2: 使用最小版本，逐个添加**

复制 `main_最小版.py` 为 `main_test.py`，然后逐个取消注释模块。

## 🔴 常见问题模块

### 嫌疑人 A: mqtt_api.py（网络阻塞）

**症状**：导入时卡住，最后一行是 "正在加载 MQTT API 模块..."

**常见原因**：
```python
# mqtt_api.py 错误示范
import paho.mqtt.client as mqtt

client = mqtt.Client()
# 🛑 致命错误：在文件顶层直接连接，且没有超时设置
client.connect("127.0.0.1", 1883, 60)  # 如果 MQTT 服务器连不上，这里会卡死
```

**解决方法**：
- 把连接代码放到 `@app.on_event("startup")` 里
- 或者放到函数里，不要放在顶层

### 嫌疑人 B: unified_api.py（循环导入）

**症状**：导入时卡住，最后一行是 "正在加载 Unified API 模块..."

**常见原因**：
```python
# unified_api.py 错误示范
from main import app, DB_AVAILABLE  # 🛑 致命错误：循环导入

# main.py 导入 unified_api
# unified_api 又导入 main
# 结果：死锁
```

**解决方法**：
- 绝对不要在子模块里 `from main import app`
- 使用 `APIRouter`，在 main 里 `app.include_router(...)`
- 把共享配置放到单独的 `config.py`

### 嫌疑人 C: 其他 API 模块（数据库连接）

**症状**：导入时卡住，可能伴随数据库错误

**常见原因**：
- 模块在导入时就连接数据库
- 数据库服务未启动
- 数据库连接超时

**解决方法**：
- 使用 `main_无数据库.py` 测试
- 检查数据库服务是否运行

## 📋 检查清单

### 检查每个 API 模块文件

1. **是否有循环导入？**
   ```bash
   grep -r "from main import" backend/
   ```
   如果有结果，说明有循环导入！

2. **是否在顶层连接网络/数据库？**
   - 检查文件顶层是否有 `connect()` 调用
   - 检查是否有 `client.connect()` 等

3. **是否有阻塞操作？**
   - 检查是否有同步的网络请求
   - 检查是否有文件 I/O 操作

## 🔧 修复方案

### 方案 1: 修复循环导入

**错误**：
```python
# unified_api.py
from main import app  # ❌ 循环导入
```

**正确**：
```python
# unified_api.py
from fastapi import APIRouter

router = APIRouter()  # ✅ 使用 Router

# main.py
from unified_api import router
app.include_router(router)  # ✅ 在 main 里注册
```

### 方案 2: 修复导入时阻塞

**错误**：
```python
# mqtt_api.py
client = mqtt.Client()
client.connect("127.0.0.1", 1883)  # ❌ 导入时就连接
```

**正确**：
```python
# mqtt_api.py
client = None  # ✅ 只声明，不连接

@app.on_event("startup")
async def startup():
    global client
    client = mqtt.Client()
    client.connect("127.0.0.1", 1883)  # ✅ 在启动时连接
```
