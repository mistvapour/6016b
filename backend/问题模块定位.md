## ⚠️ 发现的真正问题

### 问题：多个 API 模块在导入时就创建系统实例

这些模块在导入时就会初始化系统，可能在初始化时连接数据库/网络：

#### 1. `unified_api.py`（第 31, 37, 43 行）
```python
semantic_manager = InteroperabilityManager()  # ⚠️ 导入时就创建
cdm_system = CDMInteropSystem()              # ⚠️ 导入时就创建
universal_system = UniversalImportSystem()   # ⚠️ 导入时就创建
```

#### 2. `cdm_api.py`（第 23 行）
```python
cdm_system = CDMInteropSystem()  # ⚠️ 导入时就创建
```

#### 3. `semantic_interop_api.py`（第 23 行）
```python
interop_manager = InteroperabilityManager()  # ⚠️ 导入时就创建
```

#### 4. `universal_import_api.py`（第 22 行）
```python
universal_system = UniversalImportSystem()  # ⚠️ 导入时就创建
```

## 🔍 问题分析

**为什么会导致卡住？**

这些系统类的 `__init__` 方法可能在初始化时：
- 连接数据库（如果数据库服务未启动，会一直等待）
- 连接网络服务（如果服务不可用，会超时等待）
- 加载大文件
- 执行其他阻塞操作

**当 `main.py` 导入这些模块时，就会触发初始化，如果初始化需要连接外部服务，就会卡住！**

## 🎯 诊断步骤

- `>>> 正在加载 Semantic API 模块...` → 问题在 `semantic_interop_api.py`

### 步骤 2: 检查系统初始化代码

检查这些文件，查看 `__init__` 方法：

```bash
# 检查 CDM 系统
grep -A 20 "def __init__" backend/cdm_system.py

# 检查 Semantic 系统
```

查找是否有：

- 文件 I/O 操作
- 其他阻塞代码

## 🔧 解决方案

### 方案 1: 延迟初始化（推荐）

不要赋值时就创建实例，改为延迟创建：

```python
# 修改前
cdm_system = CDMInteropSystem()

# 修改后
_cdm_system = None

def get_cdm_system():
    global _cdm_system
    if _cdm_system is None:
        _cdm_system = CDMInteropSystem()
    return _cdm_system
```

### 方案 2: 使用启动事件

在 FastAPI 启动时才初始化：

```python
# 在 main.py 或各 API 模块中
@app.on_event("startup")
async def startup():
    global cdm_system
    cdm_system = CDMInteropSystem()
```

### 方案 3: 添加超时

在系统初始化时添加超时设置，避免长时间等待。
