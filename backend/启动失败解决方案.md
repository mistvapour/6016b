# 🔧 解决服务启动失败的问题

## 当前状态

您看到：
```
INFO: Uvicorn running on http://127.0.0.1:8000
INFO: Started reloader process [91700] using WatchFiles
```

但服务无法访问，说明：
- ✅ **Uvicorn 服务器启动了**
- ❌ **FastAPI 应用加载失败**

1. **是否看到 `Application startup complete.`？**


### 步骤 3: 查看常见错误

## 常见错误和解决方案

### ❌ 错误 1: Application startup failed

**原因：** 应用在启动时出错

**解决方法：**
1. 查看完整错误信息
2. 检查是否有语法错误
3. 检查模块导入是否失败

### ❌ 错误 2: 模块导入失败

**错误信息类似：**
```
ModuleNotFoundError: No module named 'xxx'
ImportError: cannot import name 'xxx'
```

**解决方法：**
```bash
pip install -r requirements.txt
```

### ❌ 错误 3: 数据库连接失败

**错误信息类似：**
```
mysql.connector.errors.InterfaceError: Can't connect to MySQL server
```

**解决方法：**
- 检查数据库服务是否运行
- 检查 `.env` 文件配置
- 或者暂时禁用数据库功能

### ❌ 错误 4: 端口被占用

**错误信息类似：**
```
Address already in use
```

**解决方法：**
```bash
# 查找占用端口的进程
netstat -ano | findstr :8000

# 杀掉进程
taskkill /F /PID <进程ID>
```

## 🚀 快速修复方法

### 方法 1: 不使用 reload 启动

```bash
cd backend
source .venv/Scripts/activate
python -m uvicorn main:app --host 127.0.0.1 --port 8000
```

移除 `--reload` 参数，看看是否能启动。

### 方法 2: 查看详细日志

```bash
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --log-level debug
```

使用 `--log-level debug` 查看详细日志。

### 方法 3: 检查语法错误

```bash
python -m py_compile main.py
```
