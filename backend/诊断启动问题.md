# 🔍 main.py 启动失败问题诊断

## ❌ 已知问题

### 问题 1: Segmentation Fault 当直接运行 `python main.py`

**原因分析：**

1. **Windows + Git Bash 环境兼容性问题**
   - 在 Windows 的 Git Bash 环境中，直接运行 `python main.py` 时，`reload=True` 参数可能导致段错误
   - 这是因为文件监控机制在 Windows 上的实现问题

2. **模块导入顺序问题**
   - `main.py` 在顶层导入时尝试导入多个可选模块
   - 某些模块的导入可能在直接运行时失败

3. **uvicorn 的 reload 机制**
   - 当使用 `if __name__ == "__main__"` 直接运行时，reload 机制可能无法正常工作
   - 特别是在 Windows 环境下

## ✅ 解决方案

### 最佳实践：使用 uvicorn 命令行启动

**永远使用这种方式：**
```bash
cd backend
source .venv/Scripts/activate
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

**不要直接运行：**
```bash
python main.py  # ❌ 会导致 Segmentation fault
```

### 修复 main.py 的启动代码（可选）

如果要保留 `if __name__ == "__main__"` 块，可以这样修改：

```python
if __name__ == "__main__":
    import uvicorn
    import os
    
    print("🚀 启动MIL-STD-6016 API服务器...")
    print(f"📊 数据库状态: {'可用' if DB_AVAILABLE else '不可用'}")
    # ... 其他打印信息 ...
    
    # 在 Windows 上禁用 reload，避免 Segmentation fault
    is_windows = os.name == 'nt'
    reload_enabled = not is_windows  # 在 Windows 上禁用 reload
    
    uvicorn.run(
        app, 
        host="127.0.0.1", 
        port=8000, 
        reload=reload_enabled
    )
```

或者完全移除 reload：

```python
if __name__ == "__main__":
    import uvicorn
    print("🚀 启动MIL-STD-6016 API服务器...")
    # ... 打印信息 ...
    uvicorn.run(app, host="127.0.0.1", port=8000, reload=False)
```

## 🔧 诊断步骤

1. **检查 Python 版本**
   ```bash
   python --version
   ```

2. **检查依赖是否安装**
   ```bash
   pip list | grep uvicorn
   ```

3. **测试导入**
   ```bash
   python -c "from main import app; print('Import successful')"
   ```

4. **使用正确方式启动**
   ```bash
   python -m uvicorn main:app --host 127.0.0.1 --port 8000
   ```
