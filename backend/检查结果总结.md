## ⚠️ 潜在问题发现

### 问题 1: unified_api.py 在导入时创建系统实例

**位置：** `backend/unified_api.py` 第 29-45 行

```python
try:
    from semantic_interop_system import InteroperabilityManager
    semantic_manager = InteroperabilityManager()  # ⚠️ 导入时就创建实例
except ImportError:
    semantic_manager = None

try:
    from cdm_system import CDMInteropSystem
    cdm_system = CDMInteropSystem()  # ⚠️ 导入时就创建实例
except ImportError:
    cdm_system = None

try:
    from universal_import_system import UniversalImportSystem
    universal_system = UniversalImportSystem()  # ⚠️ 导入时就创建实例
except ImportError:
    universal_system = None
```

**问题分析：**
- 这些系统在初始化时可能会：
  - 连接数据库
  - 连接网络服务
  - 加载大文件
  - 执行其他阻塞操作

**如果这些系统初始化时需要连接外部服务，导入时就会卡住！**

## 🎯 建议的修复方案

### 方案 1: 延迟初始化（推荐）

不要赋值时就创建实例，而是设为 None，在需要时才创建：

```python
# 修改前（导入时就创建）
semantic_manager = InteroperabilityManager()

# 修改后（延迟创建）
_semantic_manager = None

def get_semantic_manager():
    global _semantic_manager
    if _semantic_manager is None:
        _semantic_manager = InteroperabilityManager()
    return _semantic_manager
```

### 方案 2: 使用启动事件

在 FastAPI 的启动事件中初始化：

```python
@app.on_event("startup")
async def startup():
    global semantic_manager
    semantic_manager = InteroperabilityManager()
```

### 方案 3: 添加超时和错误处理

在系统初始化时添加超时：

```python
import signal

def create_manager_with_timeout():
    try:
        # 设置超时
        signal.alarm(5)  # 5秒超时
        manager = InteroperabilityManager()
        signal.alarm(0)  # 取消超时
        return manager
    except:
        return None
```
