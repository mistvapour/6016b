# 🚀 在新窗口中启动后端服务

## ✅ 方法 1: 使用批处理文件

双击运行：
```
backend/在新窗口启动.bat
```

这会打开一个新的命令窗口，自动启动后端服务。

## ✅ 方法 2: 手动在新窗口启动

### 步骤：

1. **打开新的命令提示符窗口**（CMD 或 PowerShell）

2. **切换到项目目录**：
   ```cmd
   cd C:\Users\Administrator\Documents\毕业论文\代码\前端\6016-app\backend
   ```

3. **激活虚拟环境**：
   ```cmd
   .venv\Scripts\activate.bat
   ```

4. **启动服务**：
   ```cmd
   python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
   ```

## ✅ 方法 3: 在 Git Bash 中启动

```bash
cd /c/Users/Administrator/Documents/毕业论文/代码/前端/6016-app/backend
source .venv/Scripts/activate
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

## 🔍 验证服务是否启动

启动后，等待 5-10 秒，然后：

1. **查看终端窗口** - 应该看到类似这样的输出：
   ```
   INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
   INFO:     Application startup complete.
   ```

2. **访问健康检查** - 在浏览器中打开：
   - http://127.0.0.1:8000/api/health

3. **访问 API 文档** - 在浏览器中打开：
   - http://127.0.0.1:8000/docs

## ⚠️ 重要提示

- **保持终端窗口打开** - 关闭窗口会停止服务
- **不要关闭运行服务的窗口**
- **按 Ctrl+C 可以停止服务**
- **如果启动失败** - 查看终端窗口的错误信息

## 🎯 服务启动成功标志

看到以下输出说明启动成功：
```
INFO:     Started server process [进程ID]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
```

## ❌ 如果启动失败

1. **检查虚拟环境** - 确保 `.venv` 目录存在
2. **检查依赖** - 运行 `pip install -r requirements.txt`
3. **检查端口** - 确保 8000 端口未被占用
4. **查看错误信息** - 终端窗口会显示具体错误
