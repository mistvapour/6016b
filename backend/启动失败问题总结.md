# 🔍 main.py 启动失败问题总结

## ❌ 问题：Segmentation Fault

当直接运行 `python main.py` 时会出现 Segmentation fault 错误。

## 🔍 问题原因

### 1. **Windows 环境下的 reload 机制问题**

在 `main.py` 的第 704 行（修复前）：
```python
uvicorn.run(app, host="127.0.0.1", port=8000, reload=True)
```

**问题：**
- 在 Windows 环境下，特别是使用 Git Bash 时，`reload=True` 会导致文件监控机制出现问题
- 直接运行 `python main.py` 时，reload 机制与 Windows 的文件系统监控不兼容
- 这会导致 Segmentation fault（段错误）

### 2. **启动方式的差异**

- ❌ `python main.py` - 直接执行脚本，reload 机制有问题
- ✅ `python -m uvicorn main:app` - 作为模块运行，更稳定可靠

## ✅ 解决方案

### 方案 1: 已修复的代码（推荐）

我已经修改了 `main.py`，现在在 Windows 上会自动禁用 reload：

```python
import os
is_windows = os.name == 'nt'
uvicorn.run(app, host="127.0.0.1", port=8000, reload=not is_windows)
```

这样可以：
- 在 Windows 上禁用 reload，避免 Segmentation fault
- 在 Linux/Mac 上仍然可以使用 reload

### 方案 2: 使用正确的启动命令（最佳实践）

**永远使用这种方式启动后端：**

```bash
cd backend
source .venv/Scripts/activate
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

这种方式：
- ✅ 避免了 Segmentation fault
- ✅ 更符合 FastAPI/uvicorn 的最佳实践
- ✅ 在所有平台上都能正常工作

## 🎯 总结

1. **根本原因**: Windows 环境下 `reload=True` 与直接运行 `python main.py` 不兼容
2. **临时修复**: 已在代码中禁用 Windows 上的 reload
3. **最佳实践**: 使用 `python -m uvicorn` 命令启动
