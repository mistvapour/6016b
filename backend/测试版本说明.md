# 🧪 测试版本 main_test.py 使用说明

## 📋 简介

`main_test.py` 是一个简化版本的 `main.py`，用于测试和排查启动问题。

### 特点

- ✅ **不导入数据库模块** - 避免数据库连接问题
- ✅ **不导入 API 模块** - 避免模块导入错误
- ✅ **只包含基本功能** - 快速启动，验证基础配置
- ✅ **返回模拟数据** - 不需要数据库也能测试

## 🚀 启动测试版本

### 方法 1: 使用批处理文件

双击运行：
```
backend/启动测试版本.bat
```

### 方法 2: 在终端中启动

```bash
cd backend
source .venv/Scripts/activate
python -m uvicorn main_test:app --host 127.0.0.1 --port 8000 --reload
```

### 方法 3: 直接运行

```bash
cd backend
source .venv/Scripts/activate
python main_test.py
```

## 🔍 验证测试版本

启动后，访问以下端点：

1. **根路径**: http://127.0.0.1:8000
   - 应该返回服务信息

2. **健康检查**: http://127.0.0.1:8000/api/health
   - 应该返回 `{"ok": true, ...}`

3. **测试端点**: http://127.0.0.1:8000/api/test
   - 应该返回所有可用的端点列表

4. **API 文档**: http://127.0.0.1:8000/docs
   - 应该显示 Swagger UI 文档

5. **搜索测试**: http://127.0.0.1:8000/api/search?q=test
   - 应该返回模拟搜索结果

## ✅ 如果测试版本能正常启动

说明：
- ✅ FastAPI 基础配置正确
- ✅ 虚拟环境配置正确
- ✅ 依赖包安装正确
- ✅ 服务器可以正常启动

**下一步：** 可以逐步将功能添加到主版本 `main.py`

## ❌ 如果测试版本也无法启动

说明问题可能在：
- 虚拟环境配置
- FastAPI 安装
- Python 环境
- 端口占用

**需要检查：**
1. 虚拟环境是否正确激活
2. 是否安装了 fastapi 和 uvicorn
3. 端口 8000 是否被占用

## 📋 快速命令

```bash
# 启动测试版本
cd backend
source .venv/Scripts/activate
python -m uvicorn main_test:app --host 127.0.0.1 --port 8000 --reload
```

## ⚠️ 注意事项

- 测试版本使用端口 **8000**，确保没有其他服务占用
- 如果主版本服务正在运行，需要先停止
- 测试版本返回的是模拟数据，不是真实数据
