# 🚀 启动前后端服务指南

## 方式一：使用启动脚本（推荐）

### Windows 系统

双击运行 `start_services.bat` 文件，或者 PowerShell 中执行：

```powershell
.\start_services.bat
```

这个脚本会在独立的窗口中启动后端和前端服务。

### Linux/Mac 系统

```bash
chmod +x start_services.sh
./start_services.sh
```

## 方式二：使用 Python 启动脚本

```bash
python start_system.py
```

这个脚本会自动检查依赖并启动前后端服务。

## 方式三：手动启动（分别启动）

### 启动后端服务

打开一个终端窗口：

```bash
cd backend
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

后端服务将在 http://localhost:8000 启动

### 启动前端服务

打开另一个终端窗口：

```bash
cd frontend
npm run dev
```

前端服务将在 http://localhost:5173 启动

## 📋 服务地址

启动成功后，您可以通过以下地址访问：

- **前端应用**: http://localhost:5173
- **后端 API**: http://localhost:8000
- **API 文档**: http://localhost:8000/docs
- **健康检查**: http://localhost:8000/api/health

## 🔍 验证服务是否运行

### 检查后端

在浏览器中访问 http://localhost:8000/api/health

或使用命令行：

```bash
curl http://localhost:8000/api/health
```

### 检查前端

在浏览器中访问 http://localhost:5173

## ❓ 常见问题

### 1. 端口被占用

如果 8000 或 5173 端口被占用，您可以：

**修改后端端口**:
```bash
python -m uvicorn main:app --host 0.0.0.0 --port 8001 --reload
```

**修改前端端口**:
编辑 `frontend/vite.config.ts` 或使用：
```bash
npm run dev -- --port 5174
```

### 2. 后端启动失败

- 检查 Python 依赖是否安装：`pip install -r backend/requirements.txt`
- 检查是否有错误信息显示在终端

### 3. 前端启动失败

- 检查 Node.js 是否安装：`node --version`
- 检查前端依赖是否安装：`cd frontend && npm install`
- 检查是否有错误信息显示在终端

### 4. 跨域问题

后端已经配置了 CORS，允许所有来源。如果仍有问题，检查后端日志。

## 🛑 停止服务

- 如果使用独立窗口启动，直接关闭窗口
- 如果使用命令行启动，按 `Ctrl+C` 停止

## 📝 注意事项

1. 后端服务需要一些时间启动（通常 3-5 秒）
2. 前端依赖第一次安装可能需要较长时间
3. 确保防火墙允许访问 8000 和 5173 端口
