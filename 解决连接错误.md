# 🔧 解决前端连接后端错误

## ❌ 错误信息

```
GET http://localhost:8000/api/search?q=a&fuzzy=1 net::ERR_CONNECTION_REFUSED
```

**原因**: 后端服务未运行

## ✅ 解决方法

### 步骤 1: 启动后端服务

**在终端中执行以下命令：**

```bash
cd backend
source .venv/Scripts/activate
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

**或者使用批处理文件：**

双击运行 `backend/启动后端.bat`

### 步骤 2: 验证后端服务

打开新浏览器标签页，访问：
- http://localhost:8000/api/health

如果看到 JSON 响应，说明后端已启动。

### 步骤 3: 刷新前端页面

回到前端页面（http://localhost:5173），刷新后重试搜索功能。

## 📋 完整的启动流程

### 终端 1 - 启动后端

```bash
cd backend
source .venv/Scripts/activate
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

**等待看到：**
```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete.
```

### 终端 2 - 启动前端（如果还没启动）

```bash
cd frontend
npm run dev
```

## ⚠️ 重要提示

1. **后端必须先启动**，然后前端才能连接
2. **不要关闭后端服务的终端窗口**，保持服务运行
3. **使用 `Ctrl+C` 停止服务**

## 🔍 检查服务状态

### 检查后端是否运行

```bash
curl http://localhost:8000/api/health
```

或者访问浏览器：http://localhost:8000/docs

### 检查端口占用

```bash
netstat -ano | findstr ":8000"
```

## 🎯 前后端连接配置

- **前端地址**: http://localhost:5173
- **后端地址**: http://localhost:8000  
- **前端配置**: `frontend/src/App.tsx` 中已配置 `const API = "http://localhost:8000"`
- **Vite 代理**: `frontend/vite.config.ts` 中已配置代理

## 📝 常见问题

### Q: 后端启动失败？
A: 检查：
- 虚拟环境是否正确激活
- 依赖是否安装：`pip install -r backend/requirements.txt`
- 端口 8000 是否被占用

### Q: 还是连接不上？
A: 
1. 确认后端服务正在运行（能看到启动日志）
2. 检查防火墙设置
3. 尝试访问 http://localhost:8000/docs 看是否能打开 API 文档
